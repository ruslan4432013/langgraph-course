Модели эмбеддингов

Примечание: Этот обзор сосредоточен на моделях эмбеддингов для текста. Мультимодальные модели эмбеддингов в настоящее время не поддерживаются LangChain.

Обзор

Модели эмбеддингов преобразуют необработанный текст — предложение, абзац или твит — в вектор чисел фиксированной длины, который захватывает семантический смысл текста. Эти векторы позволяют машинам сравнивать и искать текст на основе смысла, а не точного совпадения слов.

На практике это означает, что тексты с похожими идеями располагаются близко друг к другу в векторном пространстве. Например, вместо точного совпадения фразы «машинное обучение» эмбеддинги могут находить документы, обсуждающие связанные концепции, даже если используются другие формулировки.

Ключевые концепции

  1. Преобразование текста в вектор: Эмбеддинги преобразуют текст в числовое векторное представление.
  2. Измерение сходства: Векторы эмбеддингов можно сравнивать с помощью простых математических операций.

Как это работает

  1. Векторизация — Модель кодирует каждую входную строку как многомерный вектор.
  2. Оценка сходства — Векторы сравниваются с помощью математических метрик для измерения степени связанности исходных текстов.

Исторический контекст

Ландшафт моделей эмбеддингов значительно изменился за последние годы. Переломным моментом стало появление в 2018 году BERT (Bidirectional Encoder Representations from Transformers) от Google. BERT применил трансформерные модели для преобразования текста в простое векторное представление, что привело к беспрецедентным результатам в различных задачах NLP.

Однако BERT не был оптимизирован для эффективной генерации эмбеддингов предложений. Это ограничение стимулировало создание SBERT (Sentence-BERT), который адаптировал архитектуру BERT для генерации семантически насыщенных эмбеддингов предложений, легко сравнимых с помощью метрик сходства, таких как косинусное сходство.

Сегодня экосистема моделей эмбеддингов разнообразна, и множество провайдеров предлагают свои собственные реализации. Для навигации в этом многообразии исследователи и практики часто обращаются к бенчмаркам, таким как Massive Text Embedding Benchmark (MTEB).

Интерфейс

LangChain предоставляет стандартный интерфейс для моделей текстовых эмбеддингов (например, OpenAI, Cohere, Hugging Face) через интерфейс Embeddings.

Доступны два основных метода:

  • embed_documents(texts: List[str]) → List[List[float]]: Создаёт эмбеддинги для списка документов.
  • embed_query(text: str) → List[float]: Создаёт эмбеддинг для одного запроса.

Примечание: Интерфейс позволяет использовать разные стратегии для эмбеддинга запросов и документов, хотя большинство провайдеров обрабатывают их одинаково на практике.

Пример использования

from langchain_openai import OpenAIEmbeddings

embeddings_model = OpenAIEmbeddings()

# Эмбеддинг нескольких документов
embeddings = embeddings_model.embed_documents(
    [
        "Hi there!",
        "Oh, hello!",
        "What's your name?",
        "My friends call me World",
        "Hello World!"
    ]
)
print(len(embeddings), len(embeddings[0]))
# Вывод: (5, 1536)

# Эмбеддинг одного запроса
query_embedding = embeddings_model.embed_query("What is the meaning of life?")

Метрики сходства

Каждый эмбеддинг — это набор координат в многомерном пространстве. Положение каждой точки отражает смысл соответствующего текста. Подобно тому, как похожие слова находятся близко друг к другу в тезаурусе, похожие концепции оказываются рядом в пространстве эмбеддингов.

Несколько метрик обычно используются для сравнения эмбеддингов:

Косинусное сходство

Измеряет косинус угла между двумя векторами.

Евклидово расстояние

Измеряет прямолинейное расстояние между двумя точками.

Скалярное произведение

Измеряет проекцию одного вектора на другой.

Пример вычисления косинусного сходства

import numpy as np

def cosine_similarity(vec1, vec2):
    dot_product = np.dot(vec1, vec2)
    norm_vec1 = np.linalg.norm(vec1)
    norm_vec2 = np.linalg.norm(vec2)
    return dot_product / (norm_vec1 * norm_vec2)

similarity = cosine_similarity(query_embedding, document_embedding)
print("Cosine Similarity:", similarity)

Совет: Выбор метрики сходства должен основываться на модели. Например, OpenAI рекомендует косинусное сходство для своих эмбеддингов.

Кэширование

Эмбеддинги можно сохранять или временно кэшировать, чтобы избежать повторных вычислений.

Кэширование эмбеддингов выполняется с помощью CacheBackedEmbeddings. Этот wrapper хранит эмбеддинги в key-value хранилище, где текст хэшируется, и хэш используется как ключ в кэше.

Основной способ инициализации CacheBackedEmbeddings — метод from_bytes_store. Он принимает следующие параметры:

  • underlying_embedder: Модель для создания эмбеддингов.
  • document_embedding_cache: Любой ByteStore для кэширования эмбеддингов документов.
  • batch_size: (опционально, по умолчанию None) Количество документов для эмбеддинга между обновлениями хранилища.
  • namespace: (опционально, по умолчанию "") Пространство имён для кэша документов. Помогает избежать коллизий (например, установите его в имя модели эмбеддингов).
  • query_embedding_cache: (опционально, по умолчанию None) ByteStore для кэширования эмбеддингов запросов, или True для использования того же хранилища, что и document_embedding_cache.

Важно:

  • Всегда устанавливайте параметр namespace, чтобы избежать коллизий при использовании разных моделей эмбеддингов.
  • CacheBackedEmbeddings по умолчанию не кэширует эмбеддинги запросов. Для включения этой функции укажите query_embedding_cache.

Пример кэширования

import time
from langchain_classic.embeddings import CacheBackedEmbeddings
from langchain_classic.storage import LocalFileStore
from langchain_core.vectorstores import InMemoryVectorStore

# Создайте базовую модель эмбеддингов
underlying_embeddings = ...  # например, OpenAIEmbeddings(), HuggingFaceEmbeddings() и т.д.

# Хранилище сохраняет эмбеддинги в локальной файловой системе
# Это не для production-использования, но полезно для локальной разработки
store = LocalFileStore("./cache/")

cached_embedder = CacheBackedEmbeddings.from_bytes_store(
    underlying_embeddings,
    store,
    namespace=underlying_embeddings.model
)

# Пример: кэширование эмбеддинга запроса
tic = time.time()
print(cached_embedder.embed_query("Hello, world!"))
print(f"Первый вызов занял: {time.time() - tic:.2f} секунд")

# Последующие вызовы используют кэш
tic = time.time()
print(cached_embedder.embed_query("Hello, world!"))
print(f"Второй вызов занял: {time.time() - tic:.2f} секунд")

В production обычно используют более надёжное постоянное хранилище, такое как база данных или облачное хранилище. См. интеграции хранилищ для доступных опций.
