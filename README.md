# Схемы состояний в LangGraph

Этот проект представляет собой учебный пример по использованию различных способов определения схем состояний (State)
в [LangGraph](https://github.com/langchain-ai/langgraph).

В LangGraph состояние — это объект, который передается между узлами графа. Вы можете определять его структуру, используя
стандартные средства Python и библиотеки валидации.

## Содержание примера

В файле `src/main.py` продемонстрированы три подхода к созданию схем состояний:

1. **TypedDict**: Простой и гибкий способ определения структуры состояния с использованием словарей Python.
2. **Dataclasses**: Использование стандартных классов данных Python (`@dataclass`).
3. **Pydantic**: Использование моделей Pydantic для строгой валидации типов и данных в реальном времени.

### Основные моменты

- **Узлы (Nodes)**: Функции, которые принимают текущее состояние и возвращают обновления.
- **Ребра (Edges)**: Маршрутизация между узлами, включая условные переходы (`decide_mood`).
- **Компиляция**: Сборка графа и его запуск с помощью `graph.invoke()`.

## Как запустить

1. **Установите зависимости**:
   ```bash
   pip install -r requirements.txt
   ```

2. **Запустите основной скрипт**:
   ```bash
   python src/main.py
   ```

*Примечание: В текущей версии `src/main.py` активен пример с `TypedDict`. Чтобы попробовать примеры с `Dataclass`
или `Pydantic`, раскомментируйте соответствующие блоки кода в файле.*

## Структура проекта

- `src/main.py` — основной файл с примерами реализации графа и схем состояний.
- `requirements.txt` — список необходимых библиотек.
- `langgraph.json` — конфигурационный файл LangGraph.

## Требования

- Python 3.13+
- langgraph
- langchain-core
- pydantic
